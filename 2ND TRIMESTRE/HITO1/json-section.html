JSON

JSON: Simplificando el Intercambio de Datos en el Mundo Digital

Introducción

JSON (JavaScript Object Notation) es un formato de intercambio de datos ligero y legible tanto para humanos como para máquinas. Se basa en un subconjunto del lenguaje de programación JavaScript, Standard ECMA-262 3rd Edition. JSON se utiliza para almacenar una colección de pares nombre-valor o una lista ordenada de valores. Es útil para serializar objetos y arrays para transmitirlos a través de la red. A diferencia de XML, JSON no utiliza una estructura de marcado completa, lo que lo hace muy fácil de analizar y generar.

Un objeto JSON puede ser:

  • Una colección de pares nombre-valor: los nombres deben ser una cadena de texto entre comillas dobles; y el valor puede ser otra cadena de texto, un array, un número, un booleano o nulo. El valor en sí puede ser otro objeto JSON.
  • Una colección ordenada: JSON se puede usar para almacenar una colección ordenada de objetos/valores. La colección es similar a un array de primitivas y un array de objetos en los lenguajes de programación.

Historia

JSON (JavaScript Object Notation) fue creado a principios de la década de 2000 por Douglas Crockford. Crockford fue el primero en especificar y popularizar el formato JSON. El acrónimo fue originado en State Software, una compañía fundada por Crockford en marzo de 2001.

El formato JSON fue definido por Douglas Crockford a finales de 2002 y dado a conocer a través de su página web. Esta página contenía la definición del formato y una implementación en Java. En poco tiempo, el uso del formato se extendió y aparecieron implementaciones para todos los lenguajes de programación.

El motivo de su éxito es su sencillez y facilidad de uso. JSON fue diseñado para ser más legible que otros formatos, facilitando a los desarrolladores la lectura y escritura de datos. Durante los últimos años, JSON ha estado reemplazando a XML y convirtiéndose en el formato de archivo predeterminado para la transferencia de datos en la web.

El formato JSON se ha publicado además en forma de norma, tanto por la IETF como por ECMA, que también han definido otros formatos derivados como I-JSON4. Las últimas versiones de las normas son las siguientes:

  • Marzo de 2015: RFC 7493: The I-JSON Message Format, editada por Tim Bray
  • Diciembre de 2017: RFC 8259: The JavaScript Object Notation (JSON) Data Interchange Format, editada por Tim Bray
  • Diciembre de 2017: ECMA 404 (2º ed): The JSON Data Interchange Syntax
JSON ha evolucionado desde su creación a principios de la década de 2000 hasta convertirse en un estándar de facto para el intercambio de datos en la web, gracias a su simplicidad, legibilidad y facilidad de uso.

Normas de Uso y Sintaxis

Las reglas de sintaxis de JSON son sencillas:

  • Un objeto JSON está rodeado por llaves {}.
  • Los pares nombre-valor se agrupan por dos puntos : y se separan por comas ,.
  • Un array comienza con un corchete izquierdo [ y termina con un corchete derecho ].
  • Las comas finales y los ceros iniciales en un número están prohibidos.
  • Los formatos octal y hexadecimal no están permitidos.
  • Cada clave dentro del JSON debe ser única y debe estar entre comillas dobles.
  • Los tipos booleanos coinciden solo con dos valores especiales: true y false y los valores NULL están representados por el literal null (sin comillas).
La biblioteca incorporada JavaScript JSON proporciona dos funciones para decodificar y codificar objetos JSON: JSON.parse() y JSON.stringify().

Ejemplo de sintaxis JSON:

                        
                        {
                            "firstName": "John",
                            "lastName": "Snow",
                            "age": 25,
                            "children": [],
                            "spouse": null,
                            "address": {
                                "street": "7504 Taylor Drive",
                                "city": "New York City",
                                "state": "New York",
                                "postalCode": "11238"
                            },
                            "phoneNumbers": [
                                {
                                    "type": "mobile",
                                    "number": "212 555-3346"
                                },
                                {
                                    "type": "fax",
                                    "number": "646 555-4567"
                                }
                            ]
                        }

En este ejemplo, los primeros dos pares de nombre y valor asignan una cadena de texto a otra cadena de texto. El tercer par nombre-valor mapea una cadena de texto “age” con un número 25. El cuarto par nombre-valor mapea una cadena de texto “children” con un array vacío []. El quinto par nombre-valor mapea una cadena de texto “spouse” con un valor nulo. El sexto par nombre-valor mapea una cadena de texto “address” con otro objeto JSON. El séptimo par nombre-valor mapea una cadena de texto “phoneNumbers” con un array de objetos JSON.

Aplicaciones en el Ámbito Empresarial

JSON se ha convertido en un componente fundamental en el desarrollo de aplicaciones empresariales debido a su facilidad de uso y su capacidad para representar datos de manera estructurada. Se utiliza en una amplia gama de aplicaciones, incluyendo servicios web, comunicación entre clientes y servidores, almacenamiento de datos en bases de datos NoSQL, configuración de APIs RESTful, entre otros. Su interoperabilidad con JavaScript lo hace particularmente adecuado para el desarrollo de aplicaciones web y móviles.

Conclusiones

En resumen, JSON ha cambiado la forma en que los datos se intercambian en el mundo digital al ofrecer una sintaxis simple y legible que facilita la transmisión y manipulación de datos. Su adopción generalizada en el desarrollo de aplicaciones web y empresariales es testimonio de su eficacia y versatilidad. Con JSON, el intercambio de datos se ha vuelto más accesible y eficiente, allanando el camino para la innovación y el progreso en el mundo digital.
